# Technical Glossary - 10X CRM

Here is a list of 10 essential technical terms used throughout the "10X CRM" project, along with their English explanations and Georgian definitions.

---

### 1. Authentication
* **English:** The process of verifying the identity of a user, typically using credentials like an email and password.
* **Georgian:** მომხმარებლის იდენტურობის დადასტურების პროცესი, როგორც წესი, ელფოსტისა და პაროლის მეშვეობით.

### 2. Session
* **English:** A temporary and interactive information exchange between a user and the web application, active until logout.
* **Georgian:** დროებითი კავშირი მომხმარებელსა და აპლიკაციას შორის, რომელიც აქტიურია სისტემიდან გამოსვლამდე (ლოგაუთამდე).

### 3. Validation
* **English:** The process of checking input data against specific rules (e.g., email format or password length) before processing.
* **Georgian:** მონაცემების შემოწმება წინასწარ გაწერილ წესებზე (მაგ. პაროლის სიგრძე), სანამ სისტემა მათ დაამუშავებს.

### 4. Fetch
* **English:** A built-in JavaScript method used to make asynchronous network requests to retrieve or send data to an external API.
* **Georgian:** JavaScript-ის ჩაშენებული მეთოდი, რომლითაც ხდება ასინქრონული რექვესთების გაგზავნა გარე სერვერიდან მონაცემების მისაღებად ან გასაგზავნად.

### 5. Endpoint
* **English:** A specific URL path on an API server where external applications can access resource resources (e.g., `/users`).
* **Georgian:** კონკრეტული URL მისამართი API სერვერზე, რომელსაც აპლიკაცია უკავშირდება რესურსების მისაღებად (მაგ. `/users`).

### 6. Request Method
* **English:** An HTTP verb (such as GET, POST, or DELETE) that indicates the desired action to be performed on a resource.
* **Georgian:** HTTP მოთხოვნის ტიპი (მაგალითად GET წასაკითხად, POST ჩასაწერად, DELETE წასაშლელად), რომელიც სერვერს ეუბნება, თუ რა ქმედება გვინდა შევასრულოთ.

### 7. JSON (JavaScript Object Notation)
* **English:** A lightweight, text-based data format used for storing and transmitting structured data between a server and a client.
* **Georgian:** მონაცემთა გაცვლის მსუბუქი, ტექსტური ფორმატი, რომელიც გამოიყენება სერვერსა და კლიენტს შორის ინფორმაციის გადასაცემად.

### 8. State
* **English:** An object or variable that holds the current data and status of the application at any given moment.
* **Georgian:** მონაცემთა მიმდინარე მდგომარეობა აპლიკაციაში დროის კონკრეტულ მომენტში (მაგალითად, კლიენტების მიმდინარე მასივი).

### 9. Event Listener
* **English:** A block of code designed to wait and "listen" for a specific user action, such as a click, change, or submit event.
* **Georgian:** კოდის ნაწილი, რომელიც "უსმენს" მომხმარებლის ქმედებებს (მაგ. ღილაკზე დაჭერას ან ფორმის გაგზავნას) და რეაგირებს მასზე.

### 10. Deployment
* **English:** The process of hosting and publishing a web application on a live server (like Vercel or Netlify) so users can access it online.
* **Georgian:** ვებ-აპლიკაციის ატვირთვა და გაშვება რეალურ სერვერზე (მაგ. Vercel-ზე), რათა ის ხელმისაწვდომი გახდეს ინტერნეტში ყველასთვის.
